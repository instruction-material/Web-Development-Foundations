# Web Development Foundations Transition Rubric

## Position in the Course Family

The intended sequence is:

`JavaScript Level 1` -> `JavaScript Level 2` -> `Web Development Foundations`

`Web Development Foundations` should be managed as the first serious local-development and publishing course in the path, not as a replacement for the JavaScript courses.

## What This Course Assumes

Students should already be reasonably comfortable with:

- variables, loops, conditionals, and functions
- arrays and objects
- DOM manipulation and event handling
- HTML and CSS page structure
- basic API ideas and simple app data flow

## What This Course Is Actually For

The main transition here is from browser-only work into real project workflow:

- folders and file structure on disk
- Git and GitHub
- npm and package management
- local servers and build scripts
- back-end basics
- databases in full projects
- deployment and hosting
- environment variables, logs, and operational thinking

## How To Manage Overlap

Some review of HTML, CSS, DOM, APIs, and data is normal, but it should stay brief and should support the move into real project workflow. If most of the course starts looking like repeated `JavaScript Level 2` browser work, the course is drifting away from its purpose.

## Instructor Decision Rule

Students who completed `JavaScript Level 2` successfully should still start `Web Development Foundations` at the beginning. Strong students may move faster through the early setup and workflow sections, but they should not skip them entirely because those workflow habits are part of the point of the course.
